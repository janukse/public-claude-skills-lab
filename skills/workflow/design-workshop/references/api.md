# API 설계 가이드

## 설계 초안 템플릿

```markdown
## API 설계 초안: <주제>

### 1. API 개요
- 목적, 대상 소비자, 인증 방식

### 2. 엔드포인트 목록
| Method | Path | 설명 | 인증 |
|--------|------|------|------|

### 3. 요청/응답 스키마
- 각 엔드포인트별 요청 본문, 응답 형식, 상태 코드

### 4. 에러 핸들링
| 에러 코드 | 상황 | 응답 형식 |
|-----------|------|-----------|

### 5. 페이지네이션/필터링
- 목록 API의 페이지네이션, 정렬, 필터 전략

### 6. 버전 관리
- API 버전 전략
```

## 검토 관점 가이드

| 검토 관점 | 핵심 질문 |
|-----------|----------|
| 일관성 | 네이밍, 응답 형식, 에러 코드가 일관적인가? |
| 사용성 | 소비자 입장에서 직관적이고 사용하기 쉬운가? |
| 보안 | 인증/인가, 입력 검증, rate limiting이 적절한가? |
| 진화 가능성 | 하위 호환성을 유지하며 확장할 수 있는가? |
